One of the major problems with most personalization and optimization tools, is the disconnect between the website authoring experiences and the authoring experience for targeted content. This disconnect causes undue confusion for authors and makes content maintaince a chore by requiring authors to log into multiple systems to maintain website content.

Last month, Adobe released AEM 6.2, the latest iteration of Adobe Experience Manager, Adobe's flagship Web Experience Management product. This new release comes with some exciting new features including a new integration with Adobe Target. This new itegration provides a much deeper integration between AEM and Target and allows authors to maintain both standard and targeted website content within a single interface.

Given the clear benefits of personalization and optimization and the improvements Adobe has made in improving the user interface and friendlyness of the AEM / Target integration, I expect more implementations will start leveraging both tools, especially when implementing AEM 6.2.

Seeing the Integration

While at Adobe Summit, I attended the Lab "Integrating Experience Manager with Adobe Analytics, Target and DTM", this lab showed the new integration between AEM and Target. In only one hour, we were able to set up all four products and integrate them together into a cohesive solution to personalize content. 

In this post, I'll show the process for using the integration. For more information on setting up or configuring the AEM / Target integration, please see the Summit Session documents.

1. Configure an Activity in AEM

The first step to using Adobe Target in AEM is to configure your Activity. An activity is an instance of targeting on a page. Each activity gets a name and you can configure which targeting engine, configuration and targeting type to use.

  

To create a new Activity, first select the Brand (which is a container of activities for a particular brand or site) then click the + button. If you wanted to modify an existing activity, you would select the name of activity from the dropdown.

2. Define Targets

Next, define the components to be targeted. You can define any number of components to be targeted on the page. To do so, simply select the Targeting icon on the component's edit bar.

3. Create Experiences

Once you have your componets selected for the Activity, you need to create the experiences. Experiences are content variants shown to a particular audience or are groups of users based on criteria. By default only the DEFAULT experience will be displayed when the Activity is first created.

To add an experience to target the experience, select "+ Add Experience Targeting". This will allow you to choose from among the pre-defined audiences or create a new audience to target.

You can create any number of audiences and order them in order in which they will be selected by Target.

4. Create the Experince Content

Once you've defined the experiences, select each one and update the content in the targeted components for the experience. You can change content, images or anything else you can edit within an AEM component.

You can quckly switch between experiences in authoring mode by selecting the experience name on the right to see how the content appears in each experience.
